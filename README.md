## Installation

```bash
$ pnpm install
```

## Running the app

```bash
# development
$ pnpm run start

# watch mode
$ pnpm run start:dev

# production mode
$ pnpm run start:prod
```

## Test

```bash

$ pnpm run test

# e2e tests
$ pnpm run test:e2e

# test coverage

# Comandos Prisma en NestJS

Este README proporciona una guía rápida sobre cómo utilizar los comandos de Prisma en un proyecto NestJS con el fin de gestionar la base de datos de manera eficiente.

## Generar cliente de Prisma

```bash
npx prisma generate
Este comando genera el cliente de Prisma a partir del esquema de base de datos definido en el archivo schema.prisma.

Crear una nueva migración
bash
Copy code
npx prisma migrate dev --name <NOMBRE_DE_LA_MIGRACION>
Crea una nueva migración con el nombre especificado. Prisma generará un archivo SQL con los cambios propuestos en el esquema de la base de datos.

Aplicar migraciones pendientes
bash
Copy code
npx prisma migrate dev
Aplica las migraciones pendientes a la base de datos. Prisma ejecutará los archivos SQL generados previamente para mantener sincronizada la estructura de la base de datos con el esquema definido en schema.prisma.

Actualizar la base de datos con el esquema actual
bash
Copy code
npx prisma db push
Compara el esquema definido en schema.prisma con la estructura actual de la base de datos y aplica los cambios necesarios para que coincidan.

Actualizar el cliente de Prisma
bash
Copy code
npx prisma generate
Si realizas cambios en el esquema de la base de datos en schema.prisma, ejecuta este comando para regenerar el cliente de Prisma y reflejar esos cambios en tu código.

Estos comandos son esenciales para el desarrollo y mantenimiento de la base de datos en tu proyecto NestJS utilizando Prisma. Para más detalles sobre Prisma, consulta su documentación oficial en Prisma Docs.
